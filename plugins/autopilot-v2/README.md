# autopilot-v2

**Status:** v1.0.0-alpha.1 (in development). v1 (`../autopilot/`) remains production until v2 reaches per-criterion parity with v1's published 94.7 baseline. See [ADR-0018](../../docs/adr/0018-rebuild-from-lessons-learned.md) and [`design/autopilot-v2.md`](../../docs/design/autopilot-v2.md) for the rebuild rationale.

High-autonomy Claude Code plugin. Proceeds by default, pauses via structured `AskUserQuestion` at categorical yellow-tier triggers, hard-blocks destructive ops via a Node hook.

## v2 in one paragraph

Triggers live in `triggers/*.json` — a single source of truth consumed by both `hooks/guard.mjs` (for tool-layer detection) and `skills/autopilot/SKILL.md` (auto-generated by `tools/gen-skill.mjs`). Skill text and hook regex cannot drift. Class-B (brief-content-only) triggers are deliberately absent; their function is subsumed by the `Assumed:` section of handback. Six v1 task-type commands collapse to one `commands/autopilot.md` with conditional addenda. The `reviewer` agent embeds the independent-audit pattern that caught a real P0 in v1's session.

## Install (dev)

This is alpha. Install from a local clone:

```bash
/plugin install ./plugins/autopilot-v2
```

Then in a Claude Code session:

```
/autopilot:autopilot fix the off-by-one in src/parse.ts
```

Or set `CLAUDE_AUTOPILOT=1` to load the skill at session start.

## Layout

```
autopilot-v2/
├── triggers/                 single source of truth (JSON registry)
├── tools/gen-skill.mjs       generate skills/autopilot/SKILL.md from registry
├── hooks/
│   ├── guard.mjs             consumes registry; inherits v1's hardened RED block list
│   └── lib/triggers.mjs      registry loader + regex compiler
├── skills/
│   ├── autopilot/SKILL.md    GENERATED — do not hand-edit
│   ├── handback/SKILL.md     unchanged from v1 (proven format)
│   └── checkpoint-format/SKILL.md  unchanged from v1
├── agents/
│   ├── verifier.md           unchanged from v1
│   └── reviewer.md           NEW — embedded audit role
├── commands/
│   ├── autopilot.md          ONE task command (was 6 in v1)
│   └── {autopilot-review,budget,checkpoint}.md  session-utility, unchanged
└── test/triggers-roundtrip.sh  architectural invariant gate
```

## Architectural invariants (must hold)

1. `bash test/triggers-roundtrip.sh` returns 0. The skill text matches the registry; every regex compiles.
2. Every yellow trigger has `class: A` or `class: A-hybrid`. Class B is structurally rejected by the registry loader.
3. `python3 ../../evals-v2/auditor.py` returns PASS on every PR diff.
4. `evals-v2/run.py` records no `UNDERRUN_SKIPPED` entries in the canonical baseline.

If any invariant breaks, the v2 contract is broken; do not ship.
